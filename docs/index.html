CamScripter microapp · v2.2.0

Your Axis camera, native in Apple Home.

Expose any Axis IP camera as a real HomeKit accessory — live video, snapshots, and a motion sensor driven by the camera's own VMD4 and Object Analytics. No hub, no bridge box, no cloud. It runs on the camera via CamScripter and HAP‑NodeJS.

Get started → See it in action

What you get

Everything is served straight from the camera — the microapp speaks the HomeKit Accessory Protocol directly.

🏠

Native HomeKit camera

Pairs from the Home app with a QR code or PIN. No Homebridge and no extra hardware.

🎥

Live SRTP video

Repackages the camera's own H.264 RTSP stream as SRTP. Zero‑transcode passthrough (-c:v copy) by default, with a libx264 fallback.

📸

Snapshots

VAPIX JPEG snapshots with Basic and Digest auth, plus short‑lived caching so HomeKit polling doesn't hammer the camera.

🚶

Two motion sensors

VMD4 and Object Analytics each publish their own HomeKit Motion Sensor, so you get separate notifications and automation triggers per detector — each with its own hold/debounce window.

🔊

OPUS audio

Audio alongside the video stream — iOS requires an advertised audio config even for video‑only cameras.

🔒

Stable identity

A persisted HAP identity so the accessory survives restarts and stays paired.

One settings screen, light & dark

The whole thing is configured from a single panel inside CamScripter — including a live pairing QR you scan straight into the Home app.

Pairing, step by step

From an unpaired camera to a live tile in the Home app — scanned straight from the CamScripter settings QR.

HomeKit flags the camera as an "Uncertified Accessory" because it isn't part of the paid MFi program — tap Add Anyway and it works normally. The accessory page shows Manufacturer Axis Communications, the real model and firmware, and HomeKit Certified: No.

In the Home app

Both cameras show up as live tiles and full‑screen streams, right alongside any other HomeKit camera.

Motion detection

The app publishes the camera's VMD4 and Object Analytics detections as two separate HomeKit Motion Sensors — “VMD Motion Sensor” and “AOA Motion Sensor” — each driving its own MotionDetected characteristic. Here a motion badge has fired in the Security view.

HomeKit is stateful, not an event log. It tracks the current state (Motion Detected / Clear) and a "last activity" timestamp — it does not count events.

To get push alerts you must enable them per sensor: long‑press it → Status & Notifications — so you can alert on VMD but not AOA, or vice versa. For a real event timeline you need HomeKit Secure Video (iCloud+) or a third‑party app like Eve. The motion_hold_s setting keeps each sensor “detected” for N seconds after its last event so a flapping detector doesn't fire a storm of notifications.

Install

Build the package, upload it to CamScripter, and pair it in the Home app.

  1. Build the package zip (see commands below).
  2. In the camera UI: CamScripter → Apps → Upload package.
  3. Open the settings UI and enter camera credentials + HomeKit options.
  4. In iOS Home → + → Add Accessory, scan the QR (or enter the PIN).
npm install
npm run zip:package      # generic CamScripter package
npm run zip:arm64        # ARTPEC-8 / 9 (aarch64) — bundles ffmpeg
npm run zip:armhf        # ARTPEC-6 / 7 (armv7)   — bundles ffmpeg
